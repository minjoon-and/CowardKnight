# 세션 이어가기

새 세션을 시작할 때 이전 세션 인계 노트를 읽고 다음 작업을 추천하는 작업입니다.

`/온보딩`이 프로젝트 전체 문서를 읽어 컨텍스트를 로드하는 무거운 작업이라면, `/세션이어가기`는 **직전 세션에서 어디까지 했고 어디서 이어갈지**만 빠르게 회복합니다.

## 절차

1. `Docs/04_Logs/Session_Handoff.md`를 읽습니다.
   - 파일이 없으면 "이전 인계 노트 없음. `/온보딩`으로 전체 컨텍스트를 로드하거나 첫 작업을 시작하세요"로 안내하고 종료.
   - 있으면 **최상단 1~2개 세션 항목**만 읽습니다 (그 이상은 무관).
2. 다음을 병렬로 실행해 인계 노트 이후 변경이 있는지 확인합니다.
   - `git log --oneline -10`
   - `git status`
3. 인계 노트의 "관련 커밋" 해시와 현재 `git log`를 비교해, **인계 노트 이후 추가된 커밋이 있는지** 확인합니다.
4. 정리해서 사용자에게 보여줍니다 (아래 출력 형식).
5. 마지막에 "**이대로 다음 시작점부터 진행할까요?**"를 묻고, 사용자 답을 받은 뒤에 실제 작업을 시작합니다.

## 출력 형식

```markdown
## 직전 세션 요약

[세션 제목 + 한 일 핵심 2~4줄. 핵심 결정도 1~2개 인용]

## 다음 시작점

[인계 노트의 "다음 세션 시작점" 그대로 또는 그 의도를 압축]

## 주의 / 미결

[인계 노트의 "주의 / 미결" + 인계 이후 변경 있으면 그것도]

## 인계 이후 변경

[인계 노트 작성 이후 추가된 커밋·미커밋 변경이 있으면 표시. 없으면 "없음"]
```

## 출력 규칙

- 짧게. 사용자가 30초 안에 컨텍스트를 회복할 수 있어야 합니다.
- 직전 세션 요약은 4줄 안쪽으로 압축합니다. 인계 노트를 그대로 옮기지 않습니다.
- "다음 시작점"이 모호하면 사용자에게 명확화 질문을 합니다.
- 인계 노트 이후 변경이 있으면 사용자에게 짚어줍니다 ("이미 커밋 N개 더 있습니다 — 의도하신 변경인가요?").
- 코드·에셋·Config·문서를 수정하지 않습니다. 컨텍스트 회복과 추천만 합니다.

## 주의

- `$ARGUMENTS`에 "전체 온보딩" 같은 추가 요청이 있으면 `/온보딩`을 권합니다 (이 스킬은 빠른 회복용).
- 인계 노트의 "다음 시작점"이 이미 완료된 작업으로 보이면 (커밋 비교로 확인 가능) 사용자에게 알리고 그 다음 일을 함께 고민합니다.
- Project Settings·World Settings 확인 같은 "사용자만 할 수 있는 작업"이 미결로 남아있으면 작업 시작 전에 확인을 요청합니다.
