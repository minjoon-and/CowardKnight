# 컨벤션 점검

코드를 CowardKnight 프로젝트 컨벤션 기준으로 점검하는 작업입니다.

## 절차

1. 기준 문서를 읽습니다.
   - `Docs/02_Architecture/Coding_Convention.md`
   - `Docs/02_Architecture/Folder_Conventions.md`
   - `Docs/03_Unreal/Unreal_Standards.md`
2. 점검 대상을 정합니다.
   - `$ARGUMENTS`에 경로가 있으면 그 파일들.
   - 없으면 `git diff HEAD --name-only`로 변경된 파일.
3. 아래 체크리스트로 점검합니다.

## 점검 체크리스트

**네이밍**

- 클래스 접두사: Actor `A`, UObject `U`, 구조체 `F`, 열거형 `E`, 인터페이스 `I` 뒤에 프로젝트 접두사 `CK`
- 파일 이름이 주요 클래스명과 일치하는가
- 에셋 이름이 Epic prefix 규칙(`BP_`, `WBP_`, `ABP_`, `DA_`, `IA_`, `IMC_` 등)을 따르는가

**폴더**

- 파일이 올바른 도메인 폴더에 있는가 (AI, AbilitySystem, Camera, Characters, Combat, Core, Data, Developer, Feedback, Input, Player, Stages, UI)
- `Source/CowardKnight/`의 도메인에 대응하는 `Content/CowardKnight/` 폴더가 있는가 (미러 규칙)
- `.md` 문서가 `Source/`나 `Content/`에 들어가 있지 않은가

**C++ 코드**

- Reflection 대상 타입에 `UCLASS`/`USTRUCT`/`UENUM`/`UINTERFACE` 사용
- UObject 멤버 참조에 `TObjectPtr<>` 사용
- 헤더 순서: Copyright → `#pragma once` → 최소 include → generated.h → forward declaration → 클래스 선언
- 헤더는 forward declaration 우선, 실제 include는 `.cpp`에
- Tick이 켜져 있다면 정당한 이유가 코드나 주석에 드러나는가 (기본은 비활성)
- 게임플레이 멤버 접근 제어가 적절한가 (기본 `protected`, 구현 세부는 `private`)
- WHAT을 설명하는 불필요한 주석이 없는가

## 출력 규칙

- 위반 항목을 `파일경로:줄번호` 형식으로 보고합니다.
- 각 항목에 무엇이 왜 위반인지, 어떻게 고치면 되는지 한 줄로 적습니다.
- 위반이 없으면 통과로 보고합니다.
- 점검만 하고, 코드를 직접 수정하지 않습니다. 수정은 사용자 승인 후 별도로 진행합니다.
