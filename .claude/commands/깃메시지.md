# 커밋 메시지 추천

CowardKnight 프로젝트의 Git 커밋 메시지를 추천하는 작업입니다.

## 절차

1. `Docs/02_Architecture/Git_Convention.md`를 읽어 규칙을 확인합니다.
2. 다음을 병렬로 실행해 변경 상태를 파악합니다.
   - `git status`
   - `git diff HEAD` (스테이징 + 비스테이징 모두)
   - `git log --oneline -10` (기존 메시지 스타일 확인)
3. 변경 내용을 분석합니다.
4. Git_Convention.md 형식으로 커밋 메시지를 추천합니다.

## 출력 규칙

- 형식:

  ```text
  type: 짧은 제목

  변경사항
  * 핵심 변경 1
  * 핵심 변경 2
  ```

- `type`은 Git_Convention.md의 타입 표(`feat`, `fix`, `docs`, `style`, `refactor`, `test`, `build`, `config`, `content`, `asset`, `chore`, `revert`)에서 고릅니다.
- 제목은 한글, 50자 안팎, 마침표 없음. Unreal 고유명사·클래스명·에셋명은 원문을 유지합니다.
- 본문 `변경사항`은 최대 5개 핵심 항목. 변경이 1개뿐이면 본문을 생략할 수 있습니다.
- 긴 설명 없이 추천 메시지를 바로 제시합니다.
- 변경 범위가 섞여 있으면(예: 코드 + 에셋 + 문서) 커밋을 나누는 방안을 먼저 제안합니다.

## 주의

- 커밋을 직접 실행하지 않습니다. 메시지만 추천합니다. 사용자가 커밋을 명시적으로 요청하면 그때 실행합니다.
- 커밋 메시지에 `Co-Authored-By` 등 AI 공동 작성자·생성 표기를 넣지 않습니다 (개발자 단독 작성).
- `$ARGUMENTS`에 범위나 파일이 주어지면 해당 범위에 한정해 분석합니다.
