# 세션 인계 노트 저장

이번 대화 세션에서 한 작업, 핵심 결정, 다음 세션이 이어갈 시작점을 `Docs/04_Logs/Session_Handoff.md`에 기록하는 작업입니다.

`/개발로그`(공식 Development Log)와는 분리된 역할입니다.

- `/개발로그` = 사용자가 정리한 공식 결정·회고 기록.
- `/세션마감` = Claude가 다음 세션이 컨텍스트를 회복할 수 있도록 남기는 인계 노트.

## 절차

1. `Docs/04_Logs/Session_Handoff.md`를 읽습니다. 파일이 없으면 새로 만듭니다 (아래 초기 헤더 사용).
2. 다음을 병렬로 실행해 세션의 실제 변경을 파악합니다.
   - `git log --oneline -15`
   - `git status`
   - `git diff HEAD --stat` (변경 파일 윤곽)
3. 대화 맥락에서 다음을 정리합니다.
   - **한 일** — 실제로 완료한 작업 (계획만 한 것은 제외)
   - **핵심 결정** — 의도가 명확한 결정 (왜 그렇게 했는지 한 줄)
   - **다음 세션 시작점** — 가장 먼저 손댈 다음 작업과 진입 경로
   - **주의 / 미결** — 사용자 확인 미완료, 임시값, 보류된 의사결정
   - **관련 커밋** — 이번 세션에서 생긴 커밋 해시 + 제목
4. 작성한 항목 초안을 사용자에게 보여주고 확인을 받습니다.
5. 확인 후 `Session_Handoff.md`의 **최상단 (헤더 바로 아래)** 에 새 항목을 삽입합니다. 기존 항목은 그대로 둡니다.

## 새 항목 형식

```markdown
## YYYY-MM-DD - 세션 제목 (한 줄 요약)

**한 일**
- ...
- ...

**핵심 결정**
- 결정: ... — 이유: ...
- 결정: ... — 이유: ...

**다음 세션 시작점**
- ...
- ...

**주의 / 미결**
- ...

**관련 커밋**
- `<hash>` <type>: <제목>
- `<hash>` <type>: <제목>

---
```

날짜는 시스템 today를 사용합니다. 같은 날 여러 번 마감할 때는 시간(`HH:mm`)을 덧붙입니다.

## 파일이 없을 때 초기 헤더

```markdown
# 세션 인계 노트

각 세션 종료 시 `/세션마감`이 새 항목을 최상단에 추가합니다.
`/세션이어가기`가 최상단부터 읽어 이전 컨텍스트를 복원합니다.

공식 개발 로그는 `Development_Log.md`를 사용합니다.

---
```

## 출력 규칙

- 초안은 5개 섹션을 모두 채웁니다. 빈 섹션이 있으면 "없음"으로 명시합니다.
- "한 일"은 결과·의도 위주. 파일 나열은 피합니다. 파일 정보는 "관련 커밋"이 대신합니다.
- "다음 세션 시작점"은 **다음 세션의 Claude가 무엇부터 손댈지 즉시 알 수 있을 만큼 구체적**으로 적습니다. "캐릭터 작업" 같은 모호한 항목은 안 됩니다 → "BP_BoltCharacter에 SKM_Manny와 Warrior_Walk_InPlace 연결, PIE에서 캐릭터·애님 확인" 수준.
- "주의 / 미결"은 사용자가 다음 세션에서 다시 확인해야 할 점을 명시.
- 한 항목당 5~10줄 안쪽으로 압축합니다. 길어지면 핵심을 놓치고 있는 것입니다.

## 주의

- 이 스킬은 Markdown 문서만 수정합니다. 코드·에셋·Config는 건드리지 않습니다.
- `$ARGUMENTS`에 추가 맥락이 있으면 (예: "캐릭터 작업 중단, 빌드 에러 해결 우선") 반영합니다.
- 항목 길이가 누적되어 파일이 너무 길어지면 사용자에게 오래된 항목 정리를 제안합니다 (직접 삭제하지 않습니다).
