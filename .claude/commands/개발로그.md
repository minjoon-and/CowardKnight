# 개발 로그 기록

이번 세션 작업을 `Docs/04_Logs/Development_Log.md`에 기록하는 작업입니다.

## 절차

1. `Docs/04_Logs/Development_Log.md`를 읽어 기존 형식과 마지막 항목을 확인합니다.
2. 이번 세션에서 무엇을 했는지 파악합니다.
   - 대화 맥락에서 수행한 작업
   - 필요하면 `git status`, `git log --oneline -5`로 변경 확인
3. 오늘 날짜(`## YYYY-MM-DD`) 항목을 기존 형식에 맞춰 작성합니다.
   - 같은 날짜 항목이 이미 있으면 그 아래에 하위 항목으로 덧붙입니다.
4. 작성한 초안을 사용자에게 보여주고 확인을 받은 뒤 문서에 추가합니다.

## 기록할 내용

- `### 제목` — 작업 묶음 이름
- 결정 사항 — 무엇을 왜 결정했는지
- 변경 — 어떤 폴더·문서·코드가 바뀌었는지
- 메모 — 임시값, 미결 사항, 다음 작업

## 출력 규칙

- 결과보다 의도와 결정을 적습니다. "수정함" 나열을 피합니다.
- 임시 튜닝값 등 임시값은 반드시 메모에 남깁니다.
- 항목은 간결하게. 너무 작은 변경을 모두 나열하지 않습니다.
- `$ARGUMENTS`로 추가 맥락이 주어지면 반영합니다.
