# 프로젝트 원칙

## 프로젝트 정체성

`CowardKnight`는 `Bolt, the Coward Knight | 겁쟁이 기사 볼트`를 만들기 위한 Unreal Engine 프로젝트입니다.

게임 장르는 2.5D 횡스크롤 로그라이크 액션입니다.

게임 컨셉은 다음과 같습니다.

> 기사단 최하위 기사 볼트가 위험에서 도망치다가, 어쩌다 보니 마을을 지키는 영웅이 된다.

## 개발 철학

이 프로젝트는 1인 개발 프로젝트입니다.

하지만 구조는 추후 대규모 게임 프로젝트로 확장될 수 있다고 가정하고 정리합니다.

빠른 임시 구현보다 공식 Unreal Engine 워크플로우, 안정적인 아키텍처, 명확한 네이밍, 예측 가능한 폴더 구조를 우선합니다.

## 참고 프로젝트

아래 Unreal 샘플 프로젝트를 아키텍처 참고 대상으로 삼습니다.

- Lyra Sample Game: 모듈형 게임플레이 구조, Enhanced Input, Gameplay Ability System, 확장 가능한 UI, 플러그인 지향 구조.
- Cropout Sample: 작고 명확한 프로젝트 구성, 관리 가능한 제작 범위, 마을 환경과 NPC 배치 방식.
- Parrot Sample: 2.5D 횡스크롤 캐릭터 이동, 전투 감각, 카메라 동작, 스테이지 흐름.

이 샘플들을 그대로 복사하지 않습니다.

샘플의 패턴을 추출한 뒤 CowardKnight의 작은 규모에 맞게 적용합니다.

## 변경 승인 규칙

코드 변경은 수정 전에 반드시 사용자에게 명시적으로 승인받아야 합니다.

요청받은 경우 별도 추가 승인 없이 가능한 작업:

- `Docs/` 문서 생성 또는 수정.
- 폴더 구조를 나타내기 위한 placeholder 파일 생성.
- 파일 읽기, 프로젝트 검색, 조사 결과 설명.

명시적 승인 없이 하면 안 되는 작업:

- C++ 파일 수정.
- Blueprint 에셋 또는 Unreal 에셋 수정.
- Config 파일 수정.
- 기존 파일 이름 변경 또는 삭제.
- 새 플러그인 또는 런타임 의존성 추가.

## LLM 에이전트 작업 규칙

다른 LLM이 이 프로젝트에서 작업할 때는 다음 순서를 지킵니다.

1. 먼저 `Docs/README.md`를 읽습니다.
2. 최근 결정 사항을 확인하기 위해 `Docs/04_Logs/Development_Log.md`를 읽습니다.
3. 코드나 에셋을 수정하기 전 사용자에게 물어봅니다.
4. 변경은 작게 유지하고, 수정 전에 어느 폴더/모듈을 건드릴지 설명합니다.
5. `Source/CowardKnight/`와 `Content/CowardKnight/`의 폴더 미러링 규칙을 따릅니다.
